The `ClementineAggregator` service orchestrates the Clementine protocol by coordinating verifiers, operators, and watchtowers for deposits, withdrawals, and system setup.

## GetNofnAggregatedXonlyPk

Returns the aggregated n-of-n x-only public key and number of verifiers.

**Request:** `Empty`

**Response:** `NofnResponse`

<ResponseField name="nofn_xonly_pk" type="bytes" required>
  Aggregated n-of-n x-only public key
</ResponseField>

<ResponseField name="num_verifiers" type="uint32" required>
  Number of verifiers participating
</ResponseField>

## Setup

Sets up the system of verifiers, watchtowers, and operators.

**Request:** `Empty`

**Response:** `VerifierPublicKeys`

<ResponseField name="verifier_public_keys" type="repeated bytes" required>
  Array of verifier public keys
</ResponseField>

**Process:**
1. Collects verifier keys from each verifier
2. Distributes these verifier keys to all verifiers
3. Collects all operator configs from each operator
4. Distributes these operator configs to all verifiers

**Usage:** Called by the clementine-backend service during initial setup.

## NewDeposit

Creates a new deposit by orchestrating the MuSig2 signing protocol across all verifiers.

**Request:** `Deposit`

<ParamField body="deposit_outpoint" type="Outpoint" required>
  User's deposit UTXO
  
  <Expandable title="Outpoint fields">
    <ResponseField name="txid" type="Txid" required>
      Transaction ID (bytes)
    </ResponseField>
    <ResponseField name="vout" type="uint32" required>
      Output index
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField body="base_deposit" type="BaseDeposit">
  New original deposit request details
  
  <Expandable title="BaseDeposit fields">
    <ResponseField name="evm_address" type="bytes" required>
      User's EVM address
    </ResponseField>
    <ResponseField name="recovery_taproot_address" type="string" required>
      User's recovery taproot address
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField body="replacement_deposit" type="ReplacementDeposit">
  Replacement deposit details
  
  <Expandable title="ReplacementDeposit fields">
    <ResponseField name="old_move_txid" type="Txid" required>
      Move-to-vault txid being replaced
    </ResponseField>
  </Expandable>
</ParamField>

**Response:** `RawSignedTx`

<ResponseField name="raw_tx" type="bytes" required>
  Raw signed move-to-vault transaction
</ResponseField>

**Process:**
1. Calls `DepositNonceGen` for every verifier
2. Aggregates nonces and sends to `DepositSign`
3. Aggregates partial signatures and sends to `DepositFinalize`
4. Calls operator to get their signatures
5. Collects partial signatures and creates the move-to-vault transaction

**Usage:** Called by clementine-backend service to initiate a deposit.

## Withdraw

Initiates a withdrawal by calling withdraw on specified operators or all operators.

**Request:** `AggregatorWithdrawalInput`

<ParamField body="withdrawal" type="WithdrawParamsWithSig" required>
  Withdrawal parameters with optional verification signature
  
  <Expandable title="WithdrawParamsWithSig fields">
    <ResponseField name="withdrawal" type="WithdrawParams" required>
      Core withdrawal parameters
    </ResponseField>
    <ResponseField name="verification_signature" type="string">
      ECDSA signature from Citrea/aggregator for authentication
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField body="operator_xonly_pks" type="repeated XOnlyPublicKeyRpc" required>
  List of operator x-only public keys to send withdrawal request to. If empty, sends to all operators.
</ParamField>

**Response:** `AggregatorWithdrawResponse`

<ResponseField name="withdraw_responses" type="repeated OperatorWithrawalResponse" required>
  Response from each operator
  
  <Expandable title="OperatorWithrawalResponse fields">
    <ResponseField name="operator_xonly_pk" type="XOnlyPublicKeyRpc" required>
      Operator's x-only public key
    </ResponseField>
    <ResponseField name="raw_tx" type="RawSignedTx">
      Raw signed payout transaction (on success)
    </ResponseField>
    <ResponseField name="error" type="string">
      Error message (on failure)
    </ResponseField>
  </Expandable>
</ResponseField>

**Usage:** Called by clementine-backend service to initiate a withdrawal.

## InternalSendTx

Sends a pre-signed transaction to the network.

**Request:** `SendTxRequest`

<ParamField body="raw_tx" type="RawSignedTx" required>
  Raw signed transaction
</ParamField>

<ParamField body="fee_type" type="FeeType" required>
  Fee payment method
  
  Options:
  - `CPFP` - Child-Pays-For-Parent
  - `RBF` - Replace-By-Fee
  - `NO_FUNDING` - No additional fee funding
</ParamField>

**Response:** `Empty`

## InternalGetEmergencyStopTx

Creates emergency stop transactions without broadcasting them.

**Request:** `GetEmergencyStopTxRequest`

<ParamField body="txids" type="repeated Txid" required>
  Array of transaction IDs to create emergency stop transactions for
</ParamField>

**Response:** `GetEmergencyStopTxResponse`

<ResponseField name="txids" type="repeated Txid" required>
  Array of transaction IDs
</ResponseField>

<ResponseField name="encrypted_emergency_stop_txs" type="repeated bytes" required>
  Array of encrypted emergency stop transactions (approximately 3 sat/vbyte fee)
</ResponseField>

<Info>
  Transactions are not broadcasted. Set `add_anchor` to true to add an anchor output for CPFP.
</Info>

---
title: "VerifierService"
description: "gRPC API reference for ClementineVerifier service methods"
---

The `ClementineVerifier` service handles verifier operations including parameter management, deposit signing, optimistic payouts, and transaction debugging.

## GetParams

Returns verifier metadata. Called once per setup to retrieve the verifier's public key.

**Request:** `Empty`

**Response:** `VerifierParams`

<ResponseField name="public_key" type="bytes" required>
  The verifier's public key
</ResponseField>

**Usage:** Called by aggregator during setup to distribute verifier public keys to all verifiers.

---

<ResponseField name="bridge_circuit_constant" type="bytes" required>

## SetOperator

Saves operator information to the verifier's database.

**Request:** `stream OperatorParams`

<ParamField body="operator_details" type="OperatorConfig">
  Operator's configuration details
  
  <Expandable title="OperatorConfig fields">
    <ResponseField name="collateral_funding_outpoint" type="Outpoint" required>
      Outpoint for collateral funding
    </ResponseField>
    <ResponseField name="xonly_pk" type="string" required>
      Operator's x-only public key
    </ResponseField>
    <ResponseField name="wallet_reimburse_address" type="string" required>
      Wallet address for reimbursements
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField body="winternitz_pubkeys" type="WinternitzPubkey">
  Winternitz public keys for each kickoff UTXO (used to commit blockhash)
  
  <Expandable title="WinternitzPubkey fields">
    <ResponseField name="digit_pubkey" type="repeated bytes" required>
      Array of digit public keys
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField body="unspent_kickoff_sig" type="SchnorrSig">
  Unspent kickoff signatures
  
  <Expandable title="SchnorrSig fields">
    <ResponseField name="schnorr_sig" type="bytes" required>
      Schnorr signature bytes
    </ResponseField>
  </Expandable>
</ParamField>

**Response:** `Empty`

**Usage:** Called by aggregator during setup to distribute operator public keys to all verifiers.

---

## SetOperatorKeys

Sets the operator's Winternitz keys and challenge ACK hashes, saving them to the database.

**Request:** `OperatorKeysWithDeposit`

<ParamField body="operator_keys" type="OperatorKeys" required>
  Operator's cryptographic keys
  
  <Expandable title="OperatorKeys fields">
    <ResponseField name="winternitz_pubkeys" type="repeated WinternitzPubkey" required>
      Winternitz public keys for each BitVM assert transaction
    </ResponseField>
    <ResponseField name="challenge_ack_digests" type="repeated ChallengeACKDigest" required>
      Hashes of preimages used to ACK watchtower challenges
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField body="deposit_params" type="DepositParams" required>
  Deposit parameters containing deposit details, actors, and security council
</ParamField>

<ParamField body="operator_xonly_pk" type="bytes" required>
  Operator's x-only public key
</ParamField>

**Response:** `Empty`

**Usage:** Called by aggregator during new deposit to distribute operator deposit information to all verifiers.

---

## NonceGen

Generates nonces for a deposit signing session.

**Request:** `NonceGenRequest`

<ParamField body="num_nonces" type="uint32" required>
  Number of nonces to generate
</ParamField>

**Response:** `stream NonceGenResponse`

<ResponseField name="first_response" type="NonceGenFirstResponse">
  Initial response containing session metadata
  
  <Expandable title="NonceGenFirstResponse fields">
    <ResponseField name="id" type="string" required>
      Nonce session ID (string representation of u128 number)
    </ResponseField>
    <ResponseField name="num_nonces" type="uint32" required>
      Number of nonces to generate
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pub_nonce" type="bytes">
  Individual public nonce (streamed after first response)
</ResponseField>

**Usage:** Called by aggregator during new deposit to generate nonces for MuSig2 signing.

---

## DepositSign

Signs deposit transactions using aggregated nonces and the verifier's secret nonce.

**Request:** `stream VerifierDepositSignParams`

<ParamField body="deposit_sign_first_param" type="DepositSignSession">
  Initial deposit sign session containing deposit parameters and nonce responses
  
  <Expandable title="DepositSignSession fields">
    <ResponseField name="deposit_params" type="DepositParams" required>
      Deposit parameters
    </ResponseField>
    <ResponseField name="nonce_gen_first_responses" type="repeated NonceGenFirstResponse" required>
      Nonce generation responses from all verifiers
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField body="agg_nonce" type="bytes">
  Aggregated nonce for signing
</ParamField>

**Response:** `stream PartialSig`

<ResponseField name="partial_sig" type="bytes" required>
  Partial signature for MuSig2 aggregation
</ResponseField>

**Usage:** Called by aggregator during new deposit for MuSig2 signing protocol.

---

## DebugTx

Retrieves the current state and history of a transaction for debugging purposes.

**Request:** `TxDebugRequest`

<ParamField body="tx_id" type="uint32" required>
  Internal transaction ID to debug
</ParamField>

**Response:** `TxDebugInfo`

<ResponseField name="id" type="uint32" required>
  Transaction internal ID
</ResponseField>

<ResponseField name="is_active" type="bool" required>
  Whether the transaction is currently active
</ResponseField>

<ResponseField name="current_state" type="string" required>
  Current state of the transaction
</ResponseField>

<ResponseField name="submission_errors" type="repeated TxDebugSubmissionError" required>
  History of submission errors
</ResponseField>

<ResponseField name="fee_payer_utxos" type="repeated TxDebugFeePayerUtxo" required>
  UTXOs used for fee payment
</ResponseField>

<ResponseField name="created_at" type="string" required>
  Timestamp of transaction creation
</ResponseField>

<ResponseField name="txid" type="Txid" required>
  Bitcoin transaction ID
</ResponseField>

<ResponseField name="fee_paying_type" type="string" required>
  Fee payment method (CPFP, RBF, etc.)
</ResponseField>

<ResponseField name="fee_payer_utxos_count" type="uint32" required>
  Total count of fee payer UTXOs
</ResponseField>

<ResponseField name="fee_payer_utxos_confirmed_count" type="uint32" required>
  Count of confirmed fee payer UTXOs
</ResponseField>

<ResponseField name="raw_tx" type="bytes" required>
  Raw transaction bytes
</ResponseField>

<ResponseField name="metadata" type="TxMetadata" required>
  Transaction metadata including deposit outpoint, operator, and transaction type
</ResponseField>

</ParamField>

<ParamField body="kickoff_id" type="KickoffId" required>

  <Expandable title="KickoffId fields">

    </ResponseField>

      Round index
    </ResponseField>
    <ResponseField name="kickoff_idx" type="uint32" required>

  </Expandable>
</ParamField>

<ResponseField name="signed_txs" type="repeated SignedTxWithType" required>

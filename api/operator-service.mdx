title: "OperatorService"
description: "gRPC API reference for ClementineOperator service methods"
---

The `ClementineOperator` service manages operator operations including withdrawals, reimbursements, deposit signing, and transaction creation.

## GetXOnlyPublicKey

Returns the operator's x-only public key.

**Request:** `Empty`

**Response:** `XOnlyPublicKeyRpc`

<ResponseField name="xonly_public_key" type="bytes" required>
  The operator's x-only public key
</ResponseField>

**Usage:** Called by aggregator during setup.

---

## GetParams

Returns operator parameters including configuration and watchtower parameters. Called once by the aggregator during setup.

**Request:** `Empty`

**Response:** `stream OperatorParams`

<ResponseField name="operator_details" type="OperatorConfig">
  Operator's configuration
  
  <Expandable title="OperatorConfig fields">
    <ResponseField name="collateral_funding_outpoint" type="Outpoint" required>
      Outpoint for collateral funding
      
      <Expandable title="Outpoint fields">
        <ResponseField name="txid" type="Txid" required>
          Transaction ID (bytes)
        </ResponseField>
        <ResponseField name="vout" type="uint32" required>
          Output index
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="xonly_pk" type="string" required>
      Operator's x-only public key
    </ResponseField>
    <ResponseField name="wallet_reimburse_address" type="string" required>
      Wallet address for reimbursements
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="winternitz_pubkeys" type="WinternitzPubkey">
  Winternitz public keys for each kickoff UTXO (used to commit blockhash)
  
  <Expandable title="WinternitzPubkey fields">
    <ResponseField name="digit_pubkey" type="repeated bytes" required>
      Array of digit public keys
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="unspent_kickoff_sig" type="SchnorrSig">
  Unspent kickoff signatures
  
  <Expandable title="SchnorrSig fields">
    <ResponseField name="schnorr_sig" type="bytes" required>
      Schnorr signature bytes
    </ResponseField>
  </Expandable>
</ResponseField>

**Usage:** Called by aggregator during setup to collect operator configuration.

## GetDepositKeys

Returns operator's deposit keys including BitVM Assert Winternitz keys and challenge ACK hashes.

**Request:** `DepositParams`

<ParamField body="deposit" type="Deposit" required>
  Deposit information
</ParamField>

<ParamField body="actors" type="Actors" required>

</ParamField>

<ParamField body="security_council" type="SecurityCouncil" required>
  Security council configuration
</ParamField>

**Response:** `OperatorKeys`

<ResponseField name="winternitz_pubkeys" type="repeated WinternitzPubkey" required>
  Winternitz public keys for each BitVM assert transaction
</ResponseField>

<ResponseField name="challenge_ack_digests" type="repeated ChallengeACKDigest" required>
  Hashes of preimages used to ACK watchtower challenges
  
  <Expandable title="ChallengeACKDigest fields">
    <ResponseField name="hash" type="bytes" required>
      Challenge ACK hash
    </ResponseField>
  </Expandable>
</ResponseField>

**Usage:** Called by aggregator during new deposit creation.

## TransferToBtcWallet

Sends the given outpoints to the operator's BTC wallet and broadcasts the transaction to the network.

**Request:** `Outpoints`

<ParamField body="outpoints" type="repeated Outpoint" required>
  Array of outpoints to transfer
  
  Each outpoint must pay to the operator's taproot address (x-only key, no merkle root). The method verifies that no outpoint is the operator's collateral.
</ParamField>

**Response:** `RawSignedTx`

<ResponseField name="raw_tx" type="bytes" required>
  Raw signed transaction that transfers the outpoints to the operator's BTC wallet address
</ResponseField>

---

## InternalWithdraw

Prepares a withdrawal if profitable and valid. Intended for operator's own use without aggregator signature.

**Request:** `WithdrawParams`

<ParamField body="withdrawal_id" type="uint32" required>
  Withdrawal ID in Citrea
</ParamField>

<ParamField body="input_signature" type="bytes" required>
  User's `SinglePlusAnyoneCanPay` signature
</ParamField>

<ParamField body="input_outpoint" type="Outpoint" required>
  User's UTXO to claim the deposit
</ParamField>

<ParamField body="output_script_pubkey" type="bytes" required>
  Withdrawal output's script pubkey (signature is only valid for this pubkey)
</ParamField>

<ParamField body="output_amount" type="uint64" required>
  Withdrawal output's amount (signature is only valid for this amount)
</ParamField>

**Response:** `RawSignedTx`

<ResponseField name="raw_tx" type="bytes" required>
  Raw signed payout transaction
</ResponseField>

<Warning>
  If automation is disabled, the withdrawal will be rejected. The withdrawal must be correct and registered in the Citrea bridge contract.
</Warning>

---

## Withdraw

Prepares a withdrawal with optional verification signature. Checks verification address if configured.

**Request:** `WithdrawParamsWithSig`

<ParamField body="withdrawal" type="WithdrawParams" required>
  Withdrawal parameters
</ParamField>

<ParamField body="verification_signature" type="string">
  Optional ECDSA signature from Citrea/aggregator for authentication
</ParamField>

**Response:** `RawSignedTx`

<ResponseField name="raw_tx" type="bytes" required>
  Raw signed payout transaction
</ResponseField>

<Info>
  If verification address is set in operator config, the signature is verified before processing the withdrawal.
</Info>

---

## GetReimbursementTxs

Determines the next step in the kickoff process and returns raw signed transactions needed for the reimbursement process.

**Request:** `Outpoint`

<ParamField body="txid" type="Txid" required>
  Transaction ID
</ParamField>

<ParamField body="vout" type="uint32" required>
  Output index
</ParamField>

**Response:** `SignedTxsWithType`

<ResponseField name="signed_txs" type="repeated SignedTxWithType" required>
  Array of raw signed transactions that the operator needs to send next
  
  <Expandable title="SignedTxWithType fields">
    <ResponseField name="transaction_type" type="GrpcTransactionId" required>
      Transaction type identifier
    </ResponseField>
    <ResponseField name="raw_tx" type="bytes" required>
      Raw transaction bytes
    </ResponseField>
  </Expandable>
</ResponseField>

<Info>
  Enables reimbursement process without automation.
</Info>

---

## InternalCreateSignedTxs

Creates and signs transactions for a given deposit and kickoff.

**Request:** `TransactionRequest`

<ParamField body="deposit_outpoint" type="Outpoint" required>
  Deposit UTXO outpoint
</ParamField>

<ParamField body="kickoff_id" type="KickoffId" required>
  Operator's kickoff ID
  
  <Expandable title="KickoffId fields">
    <ResponseField name="operator_xonly_pk" type="bytes" required>
      Operator's x-only public key
    </ResponseField>
    <ResponseField name="round_idx" type="uint32" required>
      Round index
    </ResponseField>
    <ResponseField name="kickoff_idx" type="uint32" required>
      Kickoff index
    </ResponseField>
  </Expandable>
</ParamField>

**Response:** `SignedTxsWithType`

<ResponseField name="signed_txs" type="repeated SignedTxWithType" required>
  Raw signed transactions (excludes asserts and watchtower challenge)
</ResponseField>

<Warning>
  Only used in tests.
</Warning>
